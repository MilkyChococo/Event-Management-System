# Demo Checklist

## Before demo

- app starts on local machine
- seeded accounts work
- at least 3 sample events visible
- backend tests pass
- Playwright smoke passes
- traceability matrix updated with latest status

## Suggested demo flow

1. Show backlog and explain scope discipline.
2. Show traceability matrix and test strategy.
3. Log in as student.
4. View event list and detail.
5. Register for an event.
6. Cancel the registration.
7. Log in as admin.
8. Create a new event.
9. Open attendee list.
10. Show automated test reports and CI workflow.

## Questions to prepare for

- Why was the scope limited?
- Why use unit + API + Playwright instead of only E2E?
- How do you prove duplicate registration is prevented?
- How do you prove admin access control works?
- What is your definition of done?
